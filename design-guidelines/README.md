# 프로모터스 디자인 지침서 (Design Guidelines)

이 폴더는 프로모터스(promotors.kr) UI를 설계할 때 사용한 디자인 원칙과,
**스크린샷만 보고 동일한 UI를 재현하는 방법론**을 문서화한 것입니다.

GPT, Claude Opus 등 다른 AI 모델에게 이 문서를 컨텍스트로 주입하면,
스크린샷 기반 디자인 복제 능력과 이 프로젝트의 디자인 일관성을 그대로 유지할 수 있습니다.

## 파일 구성

| 파일 | 내용 | 용도 |
|------|------|------|
| `00-screenshot-to-ui-prompt.md` | **스크린샷 → 동일 UI 재현 마스터 프롬프트** | 다른 AI 모델에 시스템 프롬프트로 그대로 붙여넣기 |
| `01-design-tokens.md` | 색상 · 타이포그래피 · 간격 · 그림자 · 라운딩 토큰 | 프로모터스 스타일로 새 화면 만들 때 참조 |
| `02-layout.md` | 히어로 분할 레이아웃 · 사선 시트 · 뷰 전환 구조 | 전체 페이지 구조 이해 |
| `03-components.md` | 버튼 · 카드 · 모달 · 캘린더 · 채팅 등 컴포넌트 레시피 | 기존 컴포넌트 재사용/확장 |
| `04-mobile.md` | 반응형 규칙 · 하단 탭바 · 앱처럼 보이는 전체화면 페이지 | 모바일 화면 작업 |

## 다른 AI 모델에 주입하는 방법

### 방법 1 — 스크린샷 복제 능력만 주입 (범용)
`00-screenshot-to-ui-prompt.md` 전체를 시스템 프롬프트(또는 대화 첫 메시지)로 붙여넣고,
그 다음에 스크린샷을 첨부하면 됩니다. 프로젝트와 무관하게 어떤 디자인이든 적용됩니다.

```
[00번 문서 내용 붙여넣기]

위 지침을 따라, 첨부한 스크린샷과 동일한 UI를 HTML/CSS로 만들어줘.
```

### 방법 2 — 프로모터스 스타일로 새 화면 제작
`01`~`04` 문서를 함께 첨부하면 이 프로젝트의 색상·간격·컴포넌트 규칙을
그대로 따르는 새 화면을 만들 수 있습니다.

```
[01~04번 문서 내용 붙여넣기]

이 디자인 시스템을 따라서 "이벤트 안내" 화면을 새로 만들어줘.
```

### 방법 3 — Claude Code / Cursor 등 코딩 에이전트
이 저장소를 열면 폴더가 자동으로 컨텍스트에 포함되므로,
"design-guidelines 폴더의 지침을 따라 작업해줘"라고 지시하면 됩니다.

## 이 문서의 유지 관리

- `style.css`의 `:root` 토큰이나 핵심 레이아웃이 바뀌면 `01`, `02` 문서를 함께 갱신하세요.
- 새 컴포넌트 패턴이 생기면 `03`에 CSS 레시피를 추가하세요.
- `00` 문서는 프로젝트와 무관한 범용 방법론이므로 수정할 일이 거의 없습니다.
